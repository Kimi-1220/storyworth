# 執筆スタジオの設計（肝①の実体: 書く/話す体験）

## 位置づけ

回答を実際に「書く / 話す」のはこのWeb執筆スタジオで行う。
LINE等の通知層（→ [line-experience.md](./line-experience.md)）が火種を作り、
スタジオが深い取材をして、小説パイプライン（→ [novel.md](./novel.md)）の素材を集める。

**LIFFは使わない。** LINEに埋め込まず、独立した自前のWebアプリにすることで
ベンダーロックを避ける。通知チャネル（LINE/メール/SMS）は将来差し替え可能。

## 着地の設計（最重要: 完成率を決める）

ターゲットは60〜90代・PC操作が苦手な層。「Webで書く」最大の壁は
**「リンクをタップしてからログイン済みの執筆画面に着くまで」の摩擦**。ここを潰す。

- **マジックリンク**: 通知に貼ったリンクをタップするだけで、パスワード入力もアプリ起動もなく
  本人のスタジオが開く（1回限り・短命トークンで本人セッションを確立）
- スマホのブラウザでそのまま完結。インストール不要・アカウント作成画面を見せない
- 文字は大きく、ボタンは少なく、1画面=1質問。迷う余地をなくす

## 入力は「書く」と「話す」が対等

一人で話すのが苦手な人も、書くのが億劫な人もいる。どちらかを「劣ったルート」に
見せない。スタジオを開くと、同格に2つの入口を出す。

```
今週の質問
「いちばん古い記憶は何ですか？」

[ 📝 書いて答える ]      [ 🎙 話して答える ]
       └──── 途中で切り替えてもOK ────┘
```

- **書く**: 大きな明朝体のエディタ。プレースホルダに書き出しの例（「わたしが最初に
  覚えているのは…」）。詰まったら「🎙話す」に切り替えられる
- **話す**: 録音ボタンひとつ。話した内容は文字起こしされ、その場で読める原稿になる。
  方言・口癖はできる限り残す（→ [novel.md](./novel.md) の「声が聞こえる」要件）。
  読み上げられたテキストはその場で手直しできる
- どちらも写真を添付できる

## 「著者気分」をどう作るか（核心）

「アンケートに回答」ではなく **「あなたの自伝の取材」** という体験にする。
語り手は“語る著者”、システムは“聞き手・編集者”。書かせるのではなく語らせ、
原稿に仕立てて差し上げる「ゴーストライター付きの著者」体験を作る。

具体レバー:

1. **役割の再定義**: 画面・コピー全体を「取材」「あなたの本」「著者」の言葉で統一する
2. **原稿に見えるUI**: フォームではなく、明朝体・紙の質感・ページの体裁。
   自分の言葉が“本の組版”で立ち上がる瞬間を作る
3. **章がリアルタイムで育つ**: 答えるそばで「第3章『川のそばの家』」が文章として形になる様子を
   見せる。「あなたの言葉が本になっていく」を可視化する
4. **著者性を与える**: 初日から表紙モックに本人の名前を“著者”として載せる（「○○の自伝」）。
   進捗は「原稿◯ページ / 本まであと◯章」と、本が分厚くなる感覚で見せる
5. **質問文を小説的にする**: 「子供の頃の家は？」ではなく
   「読者にあなたの子供の頃の家を見せてあげてください。どんな匂いがしましたか？」。
   質問自体が情景描写を誘発する（→ [novel.md](./novel.md) の五感素材要件と直結）
6. **読者からの反響**: 家族が読んで感想を送る＝著者へのファンレター。
   これを通知層から本人に届ける（最大の継続動機。→ [line-experience.md](./line-experience.md)）

## 取材フロー（1問あたり）

```
スタジオを開く（マジックリンク）
  │
  ▼
質問を提示（小説的な問いかけ） + 「書く / 話す」の選択
  │
  ▼
回答（テキスト or 録音→文字起こし） + 写真添付
  │
  ▼
やさしい深掘り（任意・1〜2問）
  「そのとき、どんな気持ちでしたか？」「それは何年ごろですか？」
  選択肢や例文で答えやすくする。しつこくしない
  │
  ▼
「本に書き加えました」 + 育っていく章のプレビュー
```

## 実装メモ

- マジックリンク認証（短命・1回限りトークン）。スマホブラウザ完結、インストール不要
- 音声: ブラウザ録音 → 文字起こし（Whisper等）。方言・言い回しを保持。その場で編集可
- 深掘りの問い・章プレビューはLLM（Claude）で生成。会話履歴と質問意図をコンテキストに渡す
- 回答（テキスト・音声の文字起こし・写真）はすべて Answer として保存し、小説素材にする
- アクセシビリティ: 大きな文字、高コントラスト、最小限のボタン、1画面1タスク
