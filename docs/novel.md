# 小説品質の設計（肝②: いかにクオリティの高い小説を書けるか）

## 課題認識

回答をそのまま時系列に並べただけの「Q&A集」では、お金を払う価値のある一冊にならない。
断片的な回答（テキスト・音声・写真）を、**読み物として面白い小説（伝記）**に仕上げる
パイプラインの品質が、顧客満足・口コミ・リピート（兄弟分の追加注文）を決める。

## 品質の定義

良い伝記小説の条件:

1. **情景が浮かぶ**: 「貧しかった」ではなく「醤油を借りに隣の家へ走った」— 具体的なエピソードと五感の描写
2. **声が聞こえる**: 語り手本人の口癖・言い回しが残っている（標準語に均しすぎない）
3. **構成がある**: 時系列の羅列ではなく、章ごとのテーマと流れ。山場がある
4. **事実が正確**: 年代・地名・人名の整合性。家族が読んで「違う」とならない
5. **写真が効いている**: エピソードに対応する写真が適切な場所に配置されている

## 執筆パイプライン

```
回答（テキスト/音声/写真）
  │
  ├─ 音声 → 文字起こし（Whisper等）。方言・言い回しは可能な限り保持
  │
  ▼
素材整理（ファクトシート化）
  - 人物・地名・年代を抽出して「事実台帳」に登録（矛盾検出にも使う）
  - エピソード単位に分割し、章テーマにタグ付け
  │
  ▼
章の執筆（LLM + 編集ルール）
  - 章ごとに素材を集めて一人称（または三人称）で執筆
  - 語り手の言い回し・口癖を意図的に残す
  - 素材が薄い章は「深掘り質問」を生成してLINE側に差し戻す ←★肝①との連動
  │
  ▼
本人・家族レビュー
  - 章が書き上がるたびにWebで読める。修正依頼・追記はLINEからでも受付
  │
  ▼
組版・入稿PDF生成 → 製本（→ printing.md）
```

## 取材（質問）側との連動 ←ここが競合との差別化

執筆パイプラインは「素材が足りない」ことを検知できる。
これをLINEの深掘り質問として差し戻すことで、**取材と執筆のループ**を作る。

- 例: 第3章「上京」の素材に年代がない → 「東京に出られたのは何年ごろですか？」をLINEで質問
- 例: 結婚式のエピソードはあるが情景描写がない → 「式場はどんな場所でしたか？」
- 例: 写真はあるがエピソードがない → 「この写真はいつ頃の、どんな場面ですか？」

## 章立て（デフォルト構成案）

質問カテゴリ（[questions_jp.md](./questions_jp.md)）がそのまま章のテーマになる:

1. 幼少期
2. 家族と育ててくれた人々
3. 成長と自分探し
4. 愛と人間関係
5. 仕事・目的・野心
6. 親であること・子育て
7. 困難な時期と回復力
8. 喜び・楽しさ・明るい側面
9. 知恵・振り返り・遺産（最終章: 家族へのメッセージ）

## 文体・人称の選択肢（注文時に選べる）

| スタイル | 説明 |
| --- | --- |
| 一人称・語りおろし | 「わたしが生まれたのは…」本人の声をそのまま活かす。デフォルト |
| 三人称・評伝風 | 「彼女が生まれたのは…」客観的で文学的な仕上がり |
| Q&A保存版 | 質問と回答をほぼそのまま収録（加工を好まない家族向け） |

## 品質管理

- **事実台帳との照合**: 生成した本文の固有名詞・年代を台帳と突き合わせ、ハルシネーションを機械的に検出
- **創作禁止の原則**: 語られていないエピソードを創作しない。描写の肉付けは本人の言葉の範囲内で
- **本人確認フロー**: 製本前に必ず全章のレビューを通す（Web + 紙ゲラ郵送オプション）
