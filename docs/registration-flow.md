# 登録の導線設計（何もない状態 → 執筆開始まで）

## このドキュメントの狙い

「サインアップから最初の1問を書き始めるまで」をできるだけ**少ないステップ・明確な主語**で
設計する。`concept.md` は贈り物モデル（子・孫 → 親・祖父母）前提だが、実際の入口は
「**だれの物語を本にするか**」で大きく二手に分かれる。ここを最初の1画面で分岐させる。

関連: [concept.md](./concept.md) / [writing-studio.md](./writing-studio.md) / [line-experience.md](./line-experience.md) / [questions-by-audience.md](./questions-by-audience.md)

---

## 核心の発見：分岐の正体は「誰が答えるか」

| | 口座の持ち主 | 本の主役 | **質問に答える人** | 状態 |
| --- | --- | --- | --- | --- |
| **A. 自分の自伝** | 本人 | 本人 | 本人 | MVP対象 |
| **B. 家族への贈り物** | 子・孫（贈り主） | 親・祖父母 | **親・祖父母** | MVP対象 |
| C. 聞き書き・追悼 | 子・孫 | 親・祖父母（故人可） | **子・孫** | 将来（質問の主語が反転するため別テンプレが必要） |

> **A と B は「答える人＝主役本人」なので、質問文がそっくり同じ（主役への二人称）になる。**
> 「あなたが初めて子どもを抱いたとき、何を思いましたか？」は自分でも親でもそのまま使える。
> 分岐しているのは**質問文ではなく、招待・通知のセットアップだけ**。
> （C だけは「お母さんはどんな人でしたか？」と主語が反転するので難易度が一段違う → 後回し）

### 設計の前提（決定事項）

- **対応範囲**: A（自分の自伝）＋ B（家族への贈り物）。C は将来拡張。
- **課金位置**: **先に購入**（ギフト前提）。執筆前に決済を挟む。
  → トレードオフ: 「最短で書き始める」原則と相反するので、**決済前に価値を体感させる仕掛け**
    （サンプルの問い・本のプレビュー見本・できあがりイメージ）を必ずLP/オンボーディングに入れる。
- **通知チャネル**: LINE 中心。ただし A（自分の自伝）では**メールも選べる**。
  通知層はベンダーロックを避け差し替え可能に保つ（→ writing-studio.md / line-experience.md）。

---

## 主語を明確にする原則（このプロダクトの肝）

質問文・コピーは「答える人＝主役」への二人称で統一する。導線の各画面で
「**いま操作しているのは誰か / 本の主役は誰か / 次に答えるのは誰か**」を常に明示する。

- 贈り物（B）では途中で**操作する人が贈り主から語り手へ移る**。この受け渡しが導線の山場。
- 画面コピーは「あなたの本」「あなたの自伝の取材」「著者」で揃える（→ writing-studio.md「著者気分」）。

---

## 入口：1画面のフォーク

決済の前に、無料で「だれの物語か」を選ぶ。これが商品構成（質問セット・本の体裁・招待の要否）を決める。

```
        だれの物語を、本にしますか？

   ┌─────────────────┐   ┌─────────────────┐
   │ 🧑 自分自身の物語   │   │ 👵 家族の物語      │
   │  自分で書く自伝     │   │  親・祖父母などへ   │
   └────────┬────────┘   └────────┬────────┘
            │ Path A               │ Path B
            ▼                      ▼
                              続柄を選ぶ
                         母 / 父 / 祖母 / 祖父 / その他
                         → questions-by-audience.md の
                           対象者別セットが初期キューに
```

続柄を選んだ瞬間に主役と質問セットが確定する（「誰に贈るか＝質問セット」）。

## Path A ― 自分の自伝

主役・著者・回答者がすべて本人。LINE 連携や招待が不要なので最短。

```
[1] フォークで「自分自身」を選ぶ
        │
[2] 著者名（本に載る名前）
    ＋（任意・スキップ可）「この本を誰に遺したい？」← トーンを決めるだけ
        │
[3] 購入（プラン選択・決済）        ← 決済前にサンプルの問い・本の見本を見せる
        │
[4] アカウント作成（メール）
    ＋ 通知チャネル選択：LINEで毎週受け取る / メールで受け取る
        │
[5] ▶ スタジオに着地。1問目が出ている状態
```

- **ペルソナ**: 終活・エンディングノート層、または日記的に残したい人。動機があり操作もできる。
  → 摩擦を最小化し「最初の数分で1行書けた」を作るのが勝ち筋。
- **ログイン**: LINE 必須ではないので**メールのマジックリンク**が基本。週次の問いは LINE でもメールでも。
- **オンボーディング**: 「書く/話すどちらでもいい」「あとから直せる」「あなたの言葉が本の組版で立ち上がる」
  を最初の1問で体感させる（→ writing-studio.md）。

---

## Path B ― 家族への贈り物

**贈り主**と**語り手**で体験が分かれる。前半は贈り主、後半（主語の受け渡し後）は語り手。

### 前半：贈り主の導線

```
[1] フォークで「家族」→ 続柄を選ぶ（母/父/祖母/祖父/その他）
        │
[2] あなた（贈り主）の情報 ＋ 語り手の呼び名
    ＋（任意）贈るメッセージ・開始日（誕生日や記念日に合わせる）
        │
[3] 購入（決済）                   ← 決済前に「親に届く画面」「できあがる本」を見せて安心感
        │
[4] 語り手への届け方
    ・LINE連携コード / QR をその場で渡す（対面）
    ・または指定日に語り手へ案内を送る
```

### 後半：語り手の導線（主語が語り手へ移る）

```
[5] 語り手：LINE公式アカウントを友だち追加 ＋ 連携コードを送る
        │
[6] マジックリンクでスタジオに着地（パスワード入力なし）
        │
[7] 温かいオンボーディング
    「これは “あなたの自伝の取材” です。○○さんからの贈り物です。」
    大きな文字・「書く / 話す どちらでもいい」・主役に祭り上げる初回メッセージ
        │
[8] ▶ 1問目
```

- **贈り主ペルソナ（30〜50代）**: 「贈り物がちゃんと届いて、機械が苦手な親でも続く」安心が欲しい。
  → 決済前に**親が受け取る画面**と**できあがる本の見本**を見せる。1問目を選ぶ or おまかせ。
- **語り手ペルソナ（60〜90代・PC苦手）**: 導線の最難関。
  → マジックリンクで着地直後にログイン済み。アカウント作成画面を見せない（→ writing-studio.md「着地の設計」）。

## ペルソナ別オンボーディングの要点

| 相手 | 心理 | オンボーディングの主眼 |
| --- | --- | --- |
| 自分の自伝（A） | 動機あり・操作できる | 摩擦ゼロで即執筆。「書く/話す」「あとで直せる」を最初の1問で体感 |
| 贈り主（B前半） | 親でも続くか不安・贈り物として成立させたい | 決済前に「親に届く画面」「できあがる本」を見せる。1問目選択 or おまかせ |
| 語り手（B後半） | 機械が苦手・自分が主役と気づいていない | マジックリンク着地・大きな文字・「あなたの自伝の取材」で著者に祭り上げる |

## データモデルへの示唆

現状の `prisma/schema.prisma` は `Storyteller`（＝答える人）単体で、贈り主や「本人/家族」の区別を持たない。
A/B を綺麗に扱うには次が要る（roadmap フェーズ3「認証」相当）。

- **アカウント（口座の持ち主）と語り手（主役）を分離**する。
  - Path A: アカウント＝語り手（同一人物）。`kind = "self"`。
  - Path B: アカウント（贈り主）が語り手を1人以上所有。`kind = "gift"`。
- `Storyteller.relationship`（self / 母 / 父 / 祖母 / 祖父 / その他）で**質問セットと主語の出し分け**。
- `Storyteller.notifyChannel`（line / email）。A はメール可、B は LINE 既定。
- 課金（先払い）を扱う購入レコード（プラン・決済状態・開始日）。

> いずれもこの導線が固まってから着手する範囲。ここでは「導線がこの分割を要求する」ことだけ記録する。

## 未解決・要検討（トレードオフ）

- **先払い × 最短執筆の両立**: 決済を挟む分、決済前の「価値の見せ方」（サンプルの問い・本のプレビュー）
  の作り込みが完成率を左右する。ここが弱いと離脱する。
- **C（聞き書き・追悼）の扱い**: 需要は確実にある（故人・要介護で本人が書けない）が、質問の主語が反転する。
  別テンプレ＆別オンボーディングが必要なので将来拡張として切り出す。
- **語り手が複数人**（両親それぞれに贈る等）: アカウントが複数 Storyteller を持つ前提で破綻しないか。
- **A→B / B→A の乗り換え**: 自分用に始めた人が親にも贈る、などの動線を後から足せる構造にしておく。
