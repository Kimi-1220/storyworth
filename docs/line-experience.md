# 通知・エンゲージメント層の設計（肝①: いかに充実した回答をもらえるか）

## 役割分担（重要）

充実した回答を集める仕事は2層に分かれる。

- **このドキュメント = 通知・関係づくりの層（LINE等）**: 質問を届け、回答にリアクションし、
  進捗を見せ、止まったら家族を巻き込んでリマインドする。「答え続けたくなる」気持ちを作る層。
- **実際に書く/話すのは [writing-studio.md](./writing-studio.md)（自前のWebスタジオ）**:
  チャットのトーク欄は短文を誘発するため、深い回答はWebスタジオで取材する。

LINEはあくまで「お知らせ役」であり、ベンダーロック回避のため将来メール/SMSにも
差し替えられる通知チャネルとして設計する（LIFFは使わない）。

## 課題認識

質問を淡々と送るだけのbotには、誰も答え続けない。
語り手（60〜90代の親・祖母）が1年間回答し続けるには、
**「聞き上手な相手に自分の話を聞いてもらえている」体験**を作る必要がある。
ただしチャットで長い回答を書かせようとすると一言で終わるため、
深掘りそのものはWebスタジオ側で行い、LINEは火種・反応・継続動機に徹する。

## 体験の構成要素

### 1. リアクション（回答への反応）

回答が来たら、必ず人間らしい反応を返す。

- 「わぁ、五右衛門風呂のお家だったんですね！」のように、回答の固有名詞・具体を拾って返す
- 写真には「素敵なお写真をありがとうございます。これは何歳ごろですか？」
- 音声には「お声で聞けて嬉しいです」
- 定型文の使い回しはNG。LLMで回答内容に即した反応を生成する

### 2. スタジオへの橋渡し（火種を深掘りにつなぐ）

深掘り（5W1H・感情・情景の聞き出し）は本来Webスタジオの仕事だが、
LINEはその火種を拾ってスタジオへ送り込む役割を担う。

- 質問プッシュには必ず「書く/話すリンク」（マジックリンク）を添える
  → タップでログイン不要のスタジオが開く（→ [writing-studio.md](./writing-studio.md)）
- 語り手がトークで短文やボイスメモを返した「軽い日」も、火種として歓迎する。
  温かく反応した上で「もっと話したくなったら、こちらで続きを話せますよ→[スタジオ]」と橋渡し
- 5W1H・感情・情景（匂い・音・色）の聞き出しはスタジオ側の取材フローで行う
  （小説の描写素材になるため。→ [novel.md](./novel.md) の素材要件と連動）
- 語り手が短い回答しかしない場合も責めない。スタジオでは選択肢や例文で答えやすくする

### 3. 小説の進捗共有（モチベーション維持）

「あなたの話が本になりつつある」ことを定期的に見せる。

- 数問回答が溜まるごとに、書き上がった章の冒頭をLINEで共有
  「第1章『川のそばの家』ができてきました。冒頭だけお見せしますね」
- 進捗率の可視化: 「本まであと32問です。3分の1まで来ました！」
- 完成した章はWebでいつでも読める（家族も読める → 家族からの感想が次の動機になる）

### 4. 家族を巻き込むリマインド

回答が止まったとき、botが語り手を直接催促し続けるのは逆効果。

- 1週間無回答 → 語り手にやさしく1回だけリマインド
- 2週間無回答 → **贈り主（子・孫）にリマインドを依頼する**
  「お母様の回答が少し止まっています。『この前の質問、私も答え気になる！』と
  ひとこと送ってもらえると再開しやすいです」
- 家族が回答に感想を送れる機能（「お父さんのこの話、初めて聞いた！」）を
  語り手にLINEで届ける → 「読まれている」実感が最大の継続動機

### 5. トーン・文体設計

- 敬語ベースだが硬すぎない、孫世代が話しかけるような温かいトーン
- 絵文字は控えめに（シニア層に読みやすく）
- 質問は1通につき1つだけ。長文を送らない
- 回答の訂正・追記はいつでも歓迎する旨を伝える（「あとから思い出したことも、いつでも送ってくださいね」）

## 会話フロー（1問あたり）

```
bot: 今週の質問です。「いちばん古い記憶は何ですか？」
     ▼ こちらから、書いても話しても答えられます
     [あなたの自伝に書き加える]  ← マジックリンク（ログイン不要でスタジオが開く）

語り手: （スタジオで書く/話して回答。短い日はトークに直接ひとことでもOK）

bot: （回答の具体を拾ったリアクション）
     第◯章が少しずつ書き上がっています。冒頭だけお見せしますね（任意）
     また来週、質問をお送りします。
```

## 実装メモ

- リアクションはLLM（Claude）で生成。会話履歴と質問の意図をコンテキストに渡す
- 深掘りの取材はスタジオ側（→ [writing-studio.md](./writing-studio.md)）。LINEは火種と反応に徹する
- マジックリンクは1回限り・短命トークンで、タップだけで本人セッションを確立（パスワード不要）
- リマインドの間隔は設定値として持つ
- 「贈り主へのリマインド依頼」のため、贈り主もLINE連携できるようにする（or メール）
- すべてのやりとり（トークの短文・スタジオの回答とも）が小説の素材として Answer に保存される
